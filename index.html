HTML

  • The head element contains information about the webpage.
  • DO not mistaken head with header...
  • The header also contains the stylesheet href command
  • The body element represents the visible content shown to the user.
  • The meta elements are not visible to the user
  • HTML is the foundation of a house, CSS decorates / paints it, and javascript opens the door to make it functional
  • HTML seems relatively straight forward (bet I'm wrong..)

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS can be used to make changes of the look and feel of the site, quicker and more efficient! :)
  • CSS is responsible for creating the site stile, as long as the correct handles are called for each element

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git config --global user.name (and email)
  • git reset --hard HEAD reverts changes to last commit
  • git status
  • git add -A
  • git commit -m "message"
  • git push origin branch-name (or "main" if pushing to main)
  • git branch (shows all local branches)
  • git checkout feature/add-css (checks out branch)
  • git pull origin main, gets you back to the main so you can branch from it if required and gets any updates.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • We have to create variables first before calling them...
  • Variables can be called by using the variable name, and then the value can be assigned to it.
  • lots can be done here
